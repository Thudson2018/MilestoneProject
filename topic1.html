Topic 1: HTML Fundamentals

HTML (HyperText Markup Language) is the foundation of all web pages. It provides the structure and content of websites, defining elements like headings, paragraphs, links, images, and more. Understanding HTML is the first step in becoming a web developer.

What is HTML?

HTML is a markup language that uses tags to define different parts of a web page. These tags tell the web browser how to display the content. HTML documents are plain text files with a .html extension that can be opened in any web browser.

Key Concept

HTML uses a system of tags enclosed in angle brackets (< >) to mark up content. Most tags come in pairs: an opening tag and a closing tag. For example: <p>This is a paragraph</p>

Basic HTML Structure

Every HTML document follows a standard structure that includes several essential elements. Here's what a basic HTML page looks like:

  • <!DOCTYPE html>: Declares the document type and HTML version
  • <html>: The root element that contains all other elements
  • <head>: Contains metadata, title, and links to stylesheets
  • <body>: Contains the visible content of the web page

Common HTML Elements

HTML provides a wide variety of elements for different types of content. Here are some of the most commonly used HTML elements:

Element Purpose Example
<h1> to <h6> Headings (h1 is largest, h6 is smallest) <h1>Main Title</h1>
<p> Paragraph of text <p>This is a paragraph.</p>
<a> Hyperlink to another page <a href="page.html">Link</a>
<img> Image <img src="image.jpg" alt="Description">
<ul> and <li> Unordered (bulleted) list <ul><li>Item</li></ul>
<ol> and <li> Ordered (numbered) list <ol><li>Item</li></ol>
<div> Container for grouping elements <div>Content here</div>

HTML Attributes

HTML elements can have attributes that provide additional information about the element. Attributes are always specified in the opening tag and usually come in name/value pairs.

  1. href: Specifies the URL for links (used with <a> tag)
  2. src: Specifies the source file for images (used with <img> tag)
  3. alt: Provides alternative text for images
  4. class: Assigns a class name for CSS styling
  5. id: Assigns a unique identifier to an element
  6. style: Adds inline CSS styling to an element

Important Note

Always include the alt attribute for images. This improves accessibility for users with screen readers and provides a description if the image fails to load. It's also important for SEO (Search Engine Optimization).

Semantic HTML

Semantic HTML uses elements that clearly describe their meaning to both the browser and the developer. Using semantic elements makes your code more readable and improves accessibility and SEO.

  • <header>: Defines a header section for a page or section
  • <nav>: Defines a navigation menu
  • <main>: Defines the main content of the page
  • <article>: Defines independent, self-contained content
  • <section>: Defines a section of related content
  • <aside>: Defines content aside from the main content
  • <footer>: Defines a footer section for a page or section

Best Practice

Use semantic HTML elements whenever possible instead of generic <div> elements. This makes your code more meaningful, improves accessibility for users with disabilities, and helps search engines better understand your content structure.

HTML Forms

HTML forms are used to collect user input. They contain form elements like text fields, checkboxes, radio buttons, and submit buttons. Forms are essential for interactive websites that need to gather information from users.

Form Element Purpose
<input type="text"> Single-line text input
<input type="email"> Email address input with validation
<input type="password"> Password input (characters are hidden)
<input type="checkbox"> Checkbox for multiple selections
<input type="radio"> Radio button for single selection
<textarea> Multi-line text input
<select> Dropdown selection menu
<button> Clickable button

Summary

HTML is the backbone of web development, providing the structure and content for all websites. By mastering HTML fundamentals, including elements, attributes, semantic markup, and forms, you'll have a solid foundation for creating well-structured web pages. Remember to always write clean, semantic HTML code that is accessible and easy to maintain.
